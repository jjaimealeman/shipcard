<task type="auto">
  <name>Task 2: Streaming JSONL reader and two-level deduplicator</name>
  <files>
    shipcard/src/parser/reader.ts
    shipcard/src/parser/deduplicator.ts
  </files>
  <action>
    Build the file discovery and streaming parser:

    1. Create `src/parser/reader.ts`:
       - `discoverJsonlFiles(projectsDir: string): AsyncGenerator<string>` — uses `glob` from `node:fs/promises` with pattern `${projectsDir}/**/*.jsonl`. IMPORTANT: use the async iterator form (`for await (const file of glob(pattern))`) — NOT the promise form which does not work. Sort is not required; files are yielded as discovered.
       - `streamJsonlFile(filePath: string): AsyncGenerator<unknown>` — uses `node:readline` createInterface on `fs.createReadStream(filePath)` with crlfDelay: Infinity. For each line: trim, skip empty, try JSON.parse and yield, catch parse errors and increment a counter. Return the skipped line count somehow — either yield a sentinel at the end, or track via a passed-in stats object. Recommendation: accept an optional `stats: { linesSkipped: number }` parameter that the caller provides, so the generator can increment it.
       - Both functions are async generators — they stream, never buffer.

    2. Create `src/parser/deduplicator.ts`:
       - `ParseResult` type: { messages: ParsedMessage[]; sessions: Map<string, { cwd: string; firstTimestamp: string; lastTimestamp: string }>; stats: { filesRead: number; linesSkipped: number } }
       - `processFile(filePath: string, seenUuids: Set<string>, stats: { linesSkipped: number }): Promise<ParsedMessage[]>` — streams one JSONL file via reader.ts, applies two-level dedup:
         - Level 1 (uuid dedup): Skip entries whose uuid is already in seenUuids. Add new uuids to seenUuids.
         - Level 2 (message.id dedup): For assistant entries sharing the same message.id, keep the one with highest output_tokens. Use a Map<string, AssistantEntry> scoped to this file.
         - For user entries: extract cwd, sessionId, timestamp. Don't create ParsedMessage — user entries are metadata only.
         - For assistant entries (after dedup): create ParsedMessage with: sessionId, timestamp, model from message.model, tokens from message.usage (map cache_creation_input_tokens to cacheCreate, cache_read_input_tokens to cacheRead, default all to 0 if missing), toolCalls extracted by filtering message.content for type === 'tool_use' and collecting their name field, cwd, isSidechain.
       - `parseAllFiles(projectsDir: string): Promise<ParseResult>` — orchestrates the full parse:
         - Discover files via discoverJsonlFiles
         - Maintain a shared seenUuids Set across all files
         - Call processFile for each file
         - Collect all ParsedMessages
         - Build sessions Map: for each ParsedMessage, track per-sessionId the cwd (from first message), firstTimestamp (min), lastTimestamp (max). Also incorporate user entry metadata (cwd from first user entry per session overrides assistant cwd if available).
         - Return ParseResult with all messages, sessions map, and stats (filesRead count, total linesSkipped)

    3. Run `npx tsc --noEmit` to verify everything compiles.

    IMPORTANT anti-patterns to avoid:
    - Do NOT buffer entire files into arrays before deduplicating — stream and dedup in the same pass
    - Do NOT use `stop_reason` as the dedup signal — use max output_tokens per message.id
    - Do NOT assume all entries have all fields — guard with optional chaining and defaults
  </action>
