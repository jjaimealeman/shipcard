  <files>

  <action>
    1. Create `src/engine/filter.ts` — date parsing and filtering:
       - `parseFilterDate(input: string): Date` — from RESEARCH.md pattern:
         - ISO date regex `/^\d{4}-\d{2}-\d{2}$/` -> `new Date(input + 'T00:00:00')` (local time)
         - `'today'` -> start of today (`new Date()` with hours/min/sec/ms set to 0)
         - Relative `/^(\d+)d$/` -> N days ago from now, set to start of day
         - Anything else: throw descriptive Error with supported formats listed
       - `filterByDateRange(messages: ParsedMessage[], since?: string, until?: string): ParsedMessage[]` — filter messages by timestamp:
         - Parse since/until strings via parseFilterDate if provided
         - Compare each message's timestamp (ISO string -> Date) against the range
         - since is inclusive (>=), until is exclusive (<) — standard range semantics
         - If neither provided, return messages unchanged
         - Return new filtered array (do not mutate input)

    2. Create `src/parser/index.ts` — barrel export:
       - Re-export parseAllFiles and ParseResult from deduplicator
       - Re-export ParsedMessage, TokenCounts from schema
       - Re-export discoverJsonlFiles, streamJsonlFile from reader (for advanced use)

    3. Create `src/engine/index.ts` — barrel export:
       - Re-export aggregate from aggregator
       - Re-export getPricing, calculateCost, formatCost from cost
       - Re-export filterByDateRange, parseFilterDate from filter
       - Re-export all types from types.ts

    4. Create `src/index.ts` — the public API:
       - `runEngine(options?: EngineOptions): Promise<AnalyticsResult>` — the single entry point:
         a. Resolve projectsDir: options.projectsDir ?? `${os.homedir()}/.claude/projects`
         b. Call parseAllFiles(projectsDir) to get ParseResult
         c. Apply date filter: if options.since or options.until provided, filter parseResult.messages via filterByDateRange. Also rebuild the sessions map from filtered messages (sessions with no remaining messages are dropped).
         d. Call getPricing() to get pricing map and version
         e. Call aggregate(filteredParseResult, pricing) to get AnalyticsResult
         f. If date range was applied, set meta.dateRange in the result
         g. Return the AnalyticsResult
       - Re-export AnalyticsResult, EngineOptions, ProjectStats, ModelStats from engine types
       - Re-export TokenCounts from parser schema
       - The AnalyticsResult is already JSON-serializable by design (all values are strings, numbers, or plain objects — no Maps, Sets, Dates, or functions). Verify this: `JSON.parse(JSON.stringify(result))` should round-trip cleanly. If any field uses Map/Set internally, convert to Record/Array before returning.

    5. Run `npx tsc --noEmit` to verify everything compiles.

    IMPORTANT:
    - Date filtering happens BEFORE aggregation, not after — this way all stats (costs, counts) reflect only the filtered window
    - The sessions map must be rebuilt after filtering to exclude sessions with no messages in the window
    - parseFilterDate uses local time (not UTC) — `new Date('2026-03-01T00:00:00')` parses as local midnight
    - AnalyticsResult must be JSON-serializable — no Maps, Sets, or class instances in the output
  </action>

  </verify>
