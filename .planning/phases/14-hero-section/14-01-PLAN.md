  <name>Task 1: Add Alpine store computed getters for today/yesterday metrics</name>
  <files>shiplog-worker/src/routes/dashboard.ts</files>
  <action>
Add the following computed getters to the Alpine.store('dashboard', {...}) object, BEFORE the sparkline generators section (~line 955):

1. **_todayDate getter** — returns YYYY-MM-DD string for TODAY in user's LOCAL timezone:
   `return new Date().toLocaleDateString('en-CA');`
   Do NOT use `toISOString().slice(0,10)` — that gives UTC, not local time.

2. **_yesterdayDate getter** — returns YYYY-MM-DD for yesterday local:
   ```
   const d = new Date();
   d.setDate(d.getDate() - 1);
   return d.toLocaleDateString('en-CA');
   ```

3. **_todayStats getter** — finds today's day object from `this.timeseries.days` (scan ALL days, not filteredDays):
   `return (this.timeseries && this.timeseries.days) ? this.timeseries.days.find(d => d.date === this._todayDate) || null : null;`

4. **_yesterdayStats getter** — same pattern for yesterday.

5. **_totalTools(day) helper** — sums all values in a day's toolCalls record:
   `return day && day.toolCalls ? Object.values(day.toolCalls).reduce((s, v) => s + v, 0) : 0;`

6. **_totalTokens(day) helper** — sums input+output+cacheCreate+cacheRead:
   `return day ? day.tokens.input + day.tokens.output + day.tokens.cacheCreate + day.tokens.cacheRead : 0;`

7. **_dir(todayVal, yesterdayVal) helper** — returns 1 (up), -1 (down), or 0 (equal).

8. **Four "today" getters** that return the formatted value for display:
   - `get todayMessages()` — `this._todayStats ? this._todayStats.messages : 0`
   - `get todaySessions()` — `this._todayStats ? this._todayStats.sessions : 0`
   - `get todayTools()` — `this._totalTools(this._todayStats)`
   - `get todayTokens()` — `this._fmtNum(this._totalTokens(this._todayStats))`

9. **Four "yesterday" getters** — same pattern with `_yesterdayStats`.

10. **Four "direction" getters** — e.g. `get dirMessages()` returns `this._dir(todayMessages raw, yesterdayMessages raw)`.

Important: todayTokens and yesterdayTokens display should use `_fmtNum` for the big number. The raw comparison for direction uses unformatted numbers.
