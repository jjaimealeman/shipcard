    Read `shipcard/src/cli/commands/sync.ts` to understand the auth config loading pattern — it reads from `~/.config/shipcard/auth.json` which has `{ token, username, workerUrl }`.

    ```typescript
    import { loadConfig } from "../config.js";
    import type { ParsedCliArgs } from "../args.js";

    // Mirror the validation from the worker — client-side validation for fast feedback
    const SLUG_MIN_LENGTH = 3;
    const SLUG_MAX_LENGTH = 50;
    const SLUG_REGEX = /^[a-z0-9][a-z0-9-]*[a-z0-9]$/;
    const SLUG_RESERVED = new Set([
      "admin", "api", "settings", "config", "dashboard",
      "billing", "sync", "auth", "webhook", "community",
      "configure", "login", "logout", "help", "support",
      "pro", "free", "upgrade", "pricing",
    ]);

    function validateSlugLocal(slug: string): string | null {
      if (slug.length < SLUG_MIN_LENGTH) return "Slug must be at least 3 characters";
      if (slug.length > SLUG_MAX_LENGTH) return "Slug must be 50 characters or fewer";
      if (!SLUG_REGEX.test(slug)) {
        return "Slug must be lowercase alphanumeric with hyphens only (no leading/trailing hyphens)";
      }
      if (SLUG_RESERVED.has(slug)) return `"${slug}" is a reserved word`;
      return null;
    }
    ```

    **Auth loading:** Read auth.json from `~/.config/shipcard/auth.json`. Use the same approach as sync.ts — check the file exists, parse JSON, extract `{ token, username, workerUrl }`. If auth not found, print "Not logged in. Run: shipcard login" and exit(1).

    **`runSlugCreate(flags, target)`:**
    - If no target (slug name), print "Usage: shipcard slug create <name> [--theme <t>] [--layout <l>]" and exit(1)
    - Validate slug name with validateSlugLocal — if invalid, print error to stderr and exit(1)
    - Build SlugConfig from flags: `{ theme: flags.theme || "catppuccin", layout: flags.layout || "classic", hide: flags.hide, heroStat: flags.heroStat }`
    - POST to `${workerUrl}/u/${username}/slugs` with `Authorization: Bearer ${token}` and JSON body `{ slug: target, config }`
    - On 201: print success line: `Created: ${workerUrl}/u/${username}/${target}`
    - On 403 (PRO required): print `"Custom slugs are a PRO feature. Upgrade at shipcard.dev/billing/checkout"` to stderr, exit(1)
    - On 409 (duplicate or limit): print the error message from response body to stderr, exit(1)
    - On other errors: print generic error to stderr, exit(1)

    **`runSlugList(flags)`:**
    - GET `${workerUrl}/u/${username}/slugs` with bearer token
    - Parse JSON response
    - If empty: print "No custom slugs. Create one: shipcard slug create <name>"
    - If slugs exist: print each slug as:
      ```
      my-dark-card    catppuccin / compact    /u/username/my-dark-card
      another-card    dracula / hero          /u/username/another-card
      ```
      Left-align slug name (padded to 20 chars), theme/layout, then URL
    - If --json flag: print raw JSON response instead

    **`runSlugDelete(flags, target)`:**
    - If no target, print "Usage: shipcard slug delete <name>" and exit(1)
    - DELETE `${workerUrl}/u/${username}/slugs/${target}` with bearer token
    - On 200: print `Deleted: ${target}`
    - On error: print error message to stderr, exit(1)

    **Main dispatch:**
    ```typescript
    export async function runSlug(
      flags: ParsedCliArgs["flags"],
      subcommand: string | undefined,
      target: string | undefined,
    ): Promise<void> {
      switch (subcommand) {
        case "create": return runSlugCreate(flags, target);
        case "list":   return runSlugList(flags);
        case "delete": return runSlugDelete(flags, target);
        default:
          printSlugHelp();
          process.exit(0);
      }
    }
    ```

      shipcard slug list
      shipcard slug delete dark-compact
