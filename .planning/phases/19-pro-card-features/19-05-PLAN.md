  <action>
    Add a "Custom Card Slugs" section to the dashboard HTML. This section goes AFTER the billing section (which was added in Phase 18-04) and BEFORE the page closing.

    **Auth approach for dashboard slug CRUD:** The dashboard is a public page. Slug CRUD needs authentication. The simplest approach that avoids building a full session system: prompt the user for their CLI bearer token in the dashboard. The token is stored in browser sessionStorage (not persisted) and sent as the Authorization header on slug API calls.

    Alternatively, the slug management section can be read-only in the dashboard (fetching via public endpoint is not possible since GET /u/:username/slugs requires auth) and direct users to use CLI for CRUD. But CONTEXT.md says "dashboard supports full CRUD."

    **Recommended approach:** Add a small "Connect CLI" flow at the top of the slug section. The user pastes their bearer token (from `~/.config/shipcard/auth.json`) into a text field. The token is stored in sessionStorage for the current tab session only. Then CRUD operations use that token. Show a "Not connected" state with instructions to run `shipcard login` first.

    **Slug section HTML (within the Alpine.js dashboard x-data):**

    Add to the Alpine store (or local x-data) the slug-related state:
    ```javascript
    slugs: [],
    slugToken: sessionStorage.getItem('shipcard_slug_token') || '',
    slugConnected: false,
    slugLoading: false,
    slugError: '',
    newSlugName: '',
    newSlugTheme: 'catppuccin',
    newSlugLayout: 'classic',
    ```

    **"Connect" mini-form:**
    - Text input for bearer token (masked as password input)
    - "Connect" button that validates the token by calling GET /u/:username/slugs
    - On success: store token in sessionStorage, set slugConnected=true, populate slugs array
    - On failure: show error "Invalid token. Run `shipcard login` to get a fresh token."

    **Slug list (visible when slugConnected=true):**
    - For each slug: show slug name, theme/layout badge, full URL (`/u/{username}/{slug}`), copy-URL button, delete button
    - Copy button: copies the full URL to clipboard using `navigator.clipboard.writeText()`
    - Delete button: calls DELETE /u/:username/slugs/:slug with bearer token, then removes from local array

    **Create form (visible when slugConnected=true AND isPro=true):**
    - Text input for slug name (live-validates on input: min 3 chars, pattern, reserved words)
    - Theme dropdown (same curated themes as configurator: catppuccin, dracula, tokyo-night, nord, gruvbox, solarized, one-dark, monokai)
    - Layout dropdown (classic, compact, hero)
    - "Create" button: POSTs to /u/:username/slugs, on success adds to list
    - Disable button if validation fails, show inline error

    **Upgrade block (visible when isPro=false):**
    - Replace the create form with a card that says:
      "Custom Slugs — PRO Feature"
      "Create clean, shareable card URLs like /u/{username}/dark-minimal"
      "Each slug saves a unique card configuration (theme, layout)"
      [Upgrade to PRO button -> /billing/checkout]
    - Style: same as existing upgrade prompts in the dashboard (from 18-04)

    **Visual design:**
    - Match existing dashboard section style: surface background, border radius, section heading
    - Use existing CSS variables: --surface, --border, --orange (for CTAs), --fg, --mid
    - Slug list items: horizontal layout with slug name bold, config as muted text, URL as monospace, buttons right-aligned
    - Create form: inputs in a row (or small grid), compact layout
    - Keep it clean and minimal — this is a utility panel, not a showcase

    **Important implementation details:**
    - The theme list for the dropdown should be hard-coded (same 8 curated themes + dark/light legacy). This matches the configurator approach.
    - The __IS_PRO__ server-side injection from Phase 18 already provides `isPro` state. Use it to conditionally show the upgrade block vs create form.
    - The __USERNAME__ server-side injection provides the username for API URL construction.
