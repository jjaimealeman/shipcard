    2. In `shipcard-worker/src/svg/renderer.ts`:
       - Add `isPro?: boolean` to the `RenderOptions` interface
       - Update `renderSvg()` to pass `options.isPro` to each layout renderer as a third/fourth argument:
         - `renderClassic(data, colors, options.isPro)` (new 3rd arg)
         - `renderCompact(data, colors, options.isPro)` (new 3rd arg)
         - `renderHero(data, colors, options.heroStat, options.isPro)` (new 4th arg)

    3. Create a shared PRO badge SVG helper — add it to EACH layout file (not a shared module, to keep layouts self-contained). The badge is a gold pill with "PRO" text:
       ```typescript
       const BADGE_COLOR = "#F59E0B";
       const BADGE_TEXT_COLOR = "#FFFFFF";
       const BADGE_WIDTH = 38;
       const BADGE_HEIGHT = 16;

       function proBadgeSvg(cardWidth: number, padding: number, badgeY: number = 12): string {
         const badgeX = cardWidth - padding - BADGE_WIDTH;
         return [
           `  <rect x="${badgeX}" y="${badgeY}" width="${BADGE_WIDTH}" height="${BADGE_HEIGHT}" rx="8" fill="${BADGE_COLOR}"/>`,
           `  <text x="${badgeX + BADGE_WIDTH / 2}" y="${badgeY + 11}" font-size="9" font-weight="700"`,
           `    text-anchor="middle" fill="${BADGE_TEXT_COLOR}"`,
           `    font-family="'Segoe UI',Ubuntu,'Helvetica Neue',Sans-Serif">PRO</text>`,
         ].join("\n");
       }
       ```

    4. In each layout file, update the exported render function signature to accept `isPro?: boolean`:
       - `renderClassic(data: CardData, colors: ThemeColors, isPro?: boolean)`
       - `renderCompact(data: CardData, colors: ThemeColors, isPro?: boolean)`
       - `renderHero(data: CardData, colors: ThemeColors, heroStat?: string, isPro?: boolean)`

    5. In each layout's render function, conditionally inject the badge SVG just BEFORE the closing `</svg>` tag:
       ```typescript
       if (isPro) {
         lines.push(proBadgeSvg(CARD_WIDTH, PADDING));
       }
       lines.push(`</svg>`);
       ```
       Make sure to find where `</svg>` is currently pushed and insert the badge before it. Each layout constructs a `lines` array and pushes `</svg>` at the end.

    The badge must be the last visual element before `</svg>` so it renders on top of everything (correct z-order by SVG paint order).
