**Project Activity chart enhancement:**

The current `buildProjectsChart()` function (around line 1668) counts "days active" per project -- it just counts how many days a project name appears in `d.projects`. With `byProject` data available, upgrade to show real metrics:

1. In the Alpine store, add a `projectSortMetric` state property defaulting to `'messages'`. This prepares for Phase 15's sort toggles but is used now for the chart.

2. Update `buildProjectsChart(days)` to use `byProject` data when available:
   ```javascript
   // If byProject data exists, aggregate real metrics
   const projectMetrics = {};
   const hasByProject = days.some(d => d.byProject);

   if (hasByProject) {
     days.forEach(d => {
       if (!d.byProject) return;
       Object.entries(d.byProject).forEach(([name, stats]) => {
         if (!projectMetrics[name]) {
           projectMetrics[name] = { messages: 0, sessions: 0, tokens: 0, costCents: 0 };
         }
         projectMetrics[name].messages += stats.messages;
         projectMetrics[name].sessions += stats.sessions;
         projectMetrics[name].tokens += stats.tokens.input + stats.tokens.output + stats.tokens.cacheCreate + stats.tokens.cacheRead;
         projectMetrics[name].costCents += stats.costCents;
       });
     });
   } else {
     // Fallback: count days active (existing behavior for old data)
     days.forEach(d => {
       (d.projects || []).forEach(p => {
         if (!projectMetrics[p]) projectMetrics[p] = { messages: 0 };
         projectMetrics[p].messages += 1; // days active as fallback
       });
     });
   }
   ```

3. Sort by `messages` (the default metric), take top 10:
   ```javascript
   const metric = 'messages'; // Phase 15 will make this dynamic
   const sorted = Object.entries(projectMetrics)
     .sort((a, b) => b[1][metric] - a[1][metric])
     .slice(0, 10);
   ```

4. Update the chart dataset label from `'Days active'` to `hasByProject ? 'Messages' : 'Days active'`.

5. Add graceful degradation: if `byProject` is missing from all days but `projects` arrays exist, fall back to the existing days-active counting. This handles old synced data (DATA-05).

6. In the "No project breakdown available" message div, keep the existing text as-is -- it correctly tells users to sync with --show-projects.
