  <action>
Rewrite the login command to use Clack for TTY mode while preserving the identical plain-text output for non-TTY.

**Import from `../clack.js`:** `intro`, `outro`, `logStep`, `logSuccess`, `logError`, `logWarn`, `createSpinner`, `note`, `isTTY`

**TTY flow (Clack walkthrough):**

1. `intro('ShipCard -- GitHub Authentication')` -- bold banner
2. `logStep('Starting GitHub device flow...')` -- progress indicator
3. In `onVerification` callback:
   - `note()` with verification URL and user code in a boxed display: `note(\`${verification.verification_uri}\n\nCode: ${verification.user_code}\`, 'Open in browser to authorize')`
   - `logStep()` mentioning expiry time
   - Still call `openUrl()` for browser open
4. Create spinner: `const s = createSpinner()`
   - `s.start('Waiting for GitHub authorization...')`
   - On success: `s.stop('GitHub authorized')`
   - On error: `s.stop('Authorization failed')` then `logError()` with details
5. After GitHub token obtained:
   - `logStep('Verifying GitHub identity...')`
   - On success: `logSuccess(\`Authenticated as ${username}\`)`
6. Token exchange:
   - `logStep('Connecting to ShipCard...')`
   - On success: `logSuccess('ShipCard token saved')`
7. `outro("You're all set! Run \`shipcard sync\` to publish your card.")`

**Non-TTY flow (pipe/MCP fallback):**

Keep EXACTLY the current behavior -- same stderr messages ("Authenticating with GitHub...", "Open this URL...", "Enter this code...", "Waiting for authorization...", etc.), same stdout output ("Logged in as {username}"), same exit codes.

**Implementation approach:**

Use `const tty = isTTY()` at the top of `runLogin()`. Branch on `tty` at each output point. The core logic (OAuth device flow, fetch calls, token exchange) stays identical -- only the output presentation changes.

**Error handling in TTY mode:**

- Timeout/expired: `logError('Device code expired.')` then `logWarn('Run \`shipcard login\` again to start a new session.')`
- Network errors: `logError(\`GitHub authentication failed: ${message}\`)`
- Token exchange errors: `logError(\`Worker token exchange failed: HTTP ${status}\`)`
- Always `process.exit(1)` on errors (unchanged)

**Critical constraints:**
- NEVER call Clack interactive prompts (confirm, text, select) in login -- login is output-only, no user input needed beyond the browser auth
- Spinner ONLY in TTY mode -- non-TTY skips spinner entirely
- The `onVerification` callback fires synchronously during auth creation -- Clack calls inside it must be safe
- Exit codes unchanged: 0 on success, 1 on any failure
