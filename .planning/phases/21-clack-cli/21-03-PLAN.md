1. `intro('ShipCard -- Delete Cloud Data')`
2. Clack confirm prompt: `const shouldDelete = await confirm(\`Delete all cloud data for ${username}? This cannot be undone.\`)`
3. If shouldDelete is false: `outro('Cancelled.')` and return (don't exit 1)

5. `outro(\`All data removed for ${username}.\`)`

- Skip Clack confirm -- proceed directly (matches current behavior where --delete doesn't ask)
- Keep all existing stderr/stdout writes unchanged

**`shipcard sync --confirm` path (TTY):**

1. `intro('ShipCard -- Sync')`
2. Spinner for analysis: `s.start('Analyzing local stats...')` / `s.stop('Stats analyzed')`
3. Keep existing sync preview output (process.stdout.write of preview table) -- this data output stays unchanged
4. Spinner for cloud sync: `s.start('Syncing to cloud...')` / `s.stop('Synced!')`
5. After success, use `logSuccess()` for card URL and `note()` for embed snippets (the markdown/HTML blocks work well in a boxed display)
6. `outro(\`View your card at ${workerUrl}/u/${username}\`)`

**`shipcard sync --confirm` path (non-TTY):**
- Keep ALL existing output identical -- same stderr progress, same stdout preview and embed snippets

**`shipcard sync` (default, no --confirm) path (TTY):**

2. Spinner for analysis (same as --confirm)
3. Keep existing preview output unchanged
4. Keep existing browser open + configurator message

- The sync preview table (Sessions, Tokens, Cost, etc.) output stays as process.stdout.write -- don't convert to Clack log calls
