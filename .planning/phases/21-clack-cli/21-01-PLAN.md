<task type="auto">
  <name>Task 1: Install @clack/prompts and create clack.ts TTY guard module</name>
  <files>shipcard/package.json, shipcard/src/cli/clack.ts</files>
  <action>
Install @clack/prompts:
```bash
cd shipcard && npm install @clack/prompts
```

Create `shipcard/src/cli/clack.ts` -- the central TTY-guard module. Every command imports Clack wrappers from here, NEVER from `@clack/prompts` directly.

Export these safe wrappers:

- `isTTY(): boolean` -- returns `process.stdout.isTTY === true`
- `intro(title: string): void` -- calls `p.intro()` only in TTY mode, else no-op
- `outro(message: string): void` -- calls `p.outro()` only in TTY mode, else no-op
- `note(message: string, title?: string): void` -- calls `p.note()` only in TTY mode, else writes plain text to stderr
- `logSuccess(msg: string): void` -- TTY: `p.log.success()`, non-TTY: stdout write
- `logStep(msg: string): void` -- TTY: `p.log.step()`, non-TTY: stderr write
- `logWarn(msg: string): void` -- TTY: `p.log.warn()`, non-TTY: stderr "Warning: " prefix
- `logError(msg: string): void` -- TTY: `p.log.error()`, non-TTY: stderr "Error: " prefix
- `confirm(message: string): Promise<boolean>` -- ONLY for TTY. Calls `p.confirm()`, checks `p.isCancel()` (exits 0 on cancel). CALLER must guard with `isTTY()` check before calling.
- `createSpinner()` -- returns `p.spinner()` instance. CALLER must guard with `isTTY()` check. In non-TTY, caller should skip spinner entirely and use logStep for progress.

All non-TTY fallback paths use the same stdout/stderr conventions as existing code: status messages to stderr, data to stdout.

Wrap all TTY Clack calls in try/catch -- on error, silently fall through to the non-TTY path (silent fallback pattern from RESEARCH.md Open Question 4).
  </action>
