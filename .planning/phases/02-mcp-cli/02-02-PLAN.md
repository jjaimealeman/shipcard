  <action>
    **Install dependencies:**
    ```bash
    cd shiplog && npm install @modelcontextprotocol/sdk zod
    ```

    **Create tool registration modules.** Each module exports a function that takes a `McpServer` instance and registers one tool. This keeps `server.ts` clean and each tool testable in isolation.

    **tools/summary.ts** — `registerSummaryTool(server: McpServer)`:
    - Import `McpServer` from `@modelcontextprotocol/sdk/server/mcp.js` (type only)
    - Import `{ z }` from `zod` (named import, not default — Zod v4)
    - Import `{ runEngine }` from `../../index.js` (.js extension required)
    - Define shared date filter schema: `since: z.string().optional().describe(...)`, `until: z.string().optional().describe(...)`
    - Register tool `shiplog:summary`:
      - title: "ShipLog Summary"
      - description: "Returns sessions, tool calls, models used, projects touched, and estimated cost from Claude Code JSONL files"
      - inputSchema: `z.object({ since, until })`
      - Handler: call `runEngine({ since, until })`, return `{ content: [{ type: 'text', text: JSON.stringify(result.summary, null, 2) }] }`
    - Use `server.registerTool()` (not deprecated `server.tool()`)

    **tools/costs.ts** — `registerCostsTool(server: McpServer)`:
    - Same imports pattern
    - Register tool `shiplog:costs`:
      - title: "ShipLog Costs"
      - description: "Returns cost breakdown by project and model from Claude Code JSONL files"
      - inputSchema: same date filter schema
      - Handler: call `runEngine({ since, until })`, return `{ content: [{ type: 'text', text: JSON.stringify({ byProject: result.byProject, byModel: result.byModel }, null, 2) }] }`

    **tools/card.ts** — `registerCardTool(server: McpServer)`:
    - Same imports pattern
    - Register tool `shiplog:card`:
      - title: "ShipLog Card Data"
      - description: "Returns raw analytics data for card generation (SVG rendering in future release)"
      - inputSchema: same date filter schema
      - Handler: call `runEngine({ since, until })`, return full AnalyticsResult as JSON text content

    **Critical rules for all tool files:**
    - NO `console.log()` — only `console.error()` if logging is needed
    - All imports use `.js` extension
    - Return type must match MCP SDK's expected `{ content: [{ type: 'text', text: string }] }`
  </action>
