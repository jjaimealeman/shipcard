  <action>
    Create three foundation modules for the CLI:

    **args.ts** — `parseCliArgs()` wrapper around `node:util.parseArgs()`:
    - Import `parseArgs` from `node:util`
    - Define options: `json` (boolean), `since` (string), `until` (string), `color` (boolean), `help` (boolean, short: 'h'), `local` (boolean — stub for Phase 3 `shiplog card --local`)
    - Use `allowPositionals: true` to capture subcommand as `positionals[0]`
    - Use `strict: false` to avoid throwing on unknown flags
    - Export a typed result: `{ command: string | undefined, flags: { json: boolean, since?: string, until?: string, color: boolean, help: boolean, local: boolean } }`
    - Destructure from null-prototype object (use spread, not hasOwnProperty)

    **config.ts** — `loadConfig()` reads `~/.shiplog.json`:
    - Interface `ShipLogConfig`: `mode?: 'compact' | 'sectioned'`, `color?: boolean`
    - Read file with `node:fs/promises`, parse JSON, return typed config
    - Missing or invalid config file returns empty object `{}` (no errors thrown)
    - Export the `ShipLogConfig` type

    **format.ts** — UTF-8 box-drawing table renderer:
    - Define BOX character constants (TL, TR, BL, BR, H, V, ML, MR, TM, BM, MM)
    - `renderTable(headers: string[], rows: string[][], options?: { color?: boolean })` — generic table renderer
    - `padRight(s: string, width: number)` — utility for column alignment
    - `formatSummary(result: AnalyticsResult, options?: { color?: boolean })` — formats summary output:
      - Overview section: sessions, total tokens (input/output/cache), models, projects, total cost, pricing version
      - Tool calls section: top tools by count
    - `formatCosts(result: AnalyticsResult, options?: { color?: boolean })` — formats costs output:
      - By-project table: project name, sessions, cost, models
      - By-model table: model name, cost, token breakdown
    - Import `AnalyticsResult` type from `../index.js` (note .js extension for Node16 ESM)
    - Plain text with UTF-8 box-drawing by default, no ANSI colors unless `color: true`
    - All .js extensions on imports (Node16 ESM resolution)
  </action>
  <verify>
    Run `npx tsc --noEmit` from `shipcard/` directory — zero type errors for the three new files.
  </verify>
  <done>
    Three foundation modules exist with correct types, exports, and .js import extensions. `parseCliArgs()` returns typed flags and subcommand. `loadConfig()` safely reads or defaults. `formatSummary()` and `formatCosts()` produce UTF-8 box-drawing tables from AnalyticsResult.
  </done>

<task type="auto">
  <name>Task 2: CLI entry point and three command handlers</name>
  <files>
    shipcard/src/cli/commands/summary.ts

    shipcard/src/cli/commands/card.ts

  <action>
    Create the command handlers and CLI entry point:

    **commands/summary.ts** — `runSummary(flags)`:
    - Call `runEngine({ since: flags.since, until: flags.until })`
    - Handle empty state: if `result.meta.filesRead === 0`, print onboarding message to stderr, exit(1)
    - Handle empty range: if `result.summary.totalSessions === 0` and date filters active, show "N sessions exist outside this range" message
    - If `flags.json`: write `JSON.stringify(result, null, 2)` to stdout
    - Else: write `formatSummary(result, { color })` to stdout
    - Exit code 2 if `result.meta.linesSkipped > 0` (print warning to stderr)
    - Exit code 0 on success

    **commands/costs.ts** — `runCosts(flags)`:
    - Same engine call and empty state handling as summary
    - If `flags.json`: write `JSON.stringify({ byProject: result.byProject, byModel: result.byModel }, null, 2)` to stdout
    - Else: write `formatCosts(result, { color })` to stdout
    - Same exit code logic as summary

    **commands/card.ts** — `runCard(flags)`:
    - Phase 2 behavior: returns raw analytics data as JSON
    - Call `runEngine()` with date filters
    - Same empty state handling
    - If `flags.json`: write full `AnalyticsResult` as JSON
    - Else: write full `AnalyticsResult` as JSON (card is always JSON in Phase 2) with a note to stderr: "Card SVG generation coming in a future release. Showing raw analytics data."
    - Same exit code logic

    **index.ts** — CLI entry point:
    - First line: `#!/usr/bin/env node`
    - Import `parseCliArgs` from `./args.js`
    - Parse args, extract command and flags
    - Define `HELP_TEXT` constant: usage line, subcommand list (summary, costs, card), global options (--json, --since, --until, --color, --help)
    - If no command or `flags.help`: print help, exit(0)
    - Load config via `loadConfig()`, merge with flags (flags override config)
    - Determine color: `shouldUseColor()` — false if `!process.stdout.isTTY`, otherwise use flag/config
    - Switch on command: 'summary' -> runSummary, 'costs' -> runCosts, 'card' -> runCard
    - Unknown command: print error + help, exit(1)
    - Wrap in try/catch: print error to stderr, exit(1)
    - All .js extensions on imports

    **Onboarding message** (shared constant or inline):
    ```
    No Claude Code session data found.

    ShipLog looks for JSONL files in: ~/.claude/projects/

    This directory is populated automatically when you use Claude Code.
    If you've used Claude Code on this machine, check that the directory exists.
    ```

    **Empty range message pattern:**
    `No sessions found in [since -> until]. N session(s) exist outside this range.`
    To get total sessions count for the empty range message, run engine without date filters first (or pass the unfiltered count). Simplest approach: run engine once with filters. If zero sessions and filters active, report it. The "sessions outside range" count requires a second unfiltered call — decide if the UX benefit is worth it. If too complex, just show "No sessions found in [range]. Try widening your date range."
  </action>
