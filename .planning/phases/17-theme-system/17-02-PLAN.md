  <name>Task 2: Rewrite card route for curated themes, BYOT, and backward compat</name>

  <action>
    Rewrite the card route handler in card.ts. The route signature stays `GET /:username`. New param handling logic:

    **Step 1: Parse all params**
    ```
    const themeParam = c.req.query("theme")     // "catppuccin", "dark", "light", etc.
    const styleParam = c.req.query("style")     // legacy: "github", "branded", "minimal"
    const layoutParam = c.req.query("layout") ?? "classic"
    const hide = c.req.queries("hide") ?? []
    // BYOT params (all optional)
    const bgParam = c.req.query("bg")
    const titleParam = c.req.query("title")
    const textParam = c.req.query("text")
    const iconParam = c.req.query("icon")
    const borderParam = c.req.query("border")
    ```

    **Step 2: Detect BYOT mode**
    If ANY of bg/title/text/icon/border params are present:
    - Check that ALL 5 are present. If partial, return error SVG: "BYOT requires all 5 color params (bg, title, text, icon, border)"
    - Validate each is a valid 6-char hex (with or without #). Use `isValidHex()` from themes. Normalize to include #.
    - Check PRO status: `await isUserPro(c.env.USER_DATA_KV, username)`. If not PRO, return error SVG: "Custom colors require ShipCard PRO — shipcard.dev/upgrade"
    - Run `validateByotContrast()` on the 5 colors. If errors, return error SVG listing each field error.
    - Construct ThemeColors: `{ bg, border, title, text, value: title, icon, footer: text }`
    - Render card with `renderCard(userData, { layout, colors, hide })` — NO caching for BYOT (skip KV read/write)

    **Step 3: Resolve curated or legacy theme**
    If NOT BYOT mode:
    - Import `CURATED_THEME_NAMES` and `resolveCuratedTheme` from svg/index.ts (re-exported from themes)
    - If themeParam is in CURATED_THEME_NAMES: resolve curated colors, use v2 cache key
    - If themeParam is "dark" or "light": legacy path — use old `resolveTheme(styleParam ?? "github", themeParam)` and old cache key
    - If themeParam is absent: default to "catppuccin" curated theme (new default for users without explicit theme)
    - If themeParam is unrecognized: fall back to "catppuccin" (graceful degradation, not error)

    **Step 4: Cache + render**
    For curated themes: use `getCardCacheV2` / `putCardCacheV2` from kv.ts
    For legacy themes: use existing `getCardCache` / `putCardCache` (unchanged)
    For BYOT: no caching — render fresh every time

    **Error SVG helper:**
    Create a local `renderErrorSvg(title: string, messages: string[]): string` function that returns a simple SVG with error text. Use a neutral dark bg (#1e1e2e), red-ish title (#ff6b6b), white text. Keep it simple -- 495x120px, same width as regular cards.

    **Imports to add:**
    - `isUserPro`, `getCardCacheV2`, `putCardCacheV2` from `../kv.js`
    - `CURATED_THEME_NAMES`, `resolveCuratedTheme`, `validateByotContrast`, `isValidHex` from `../svg/index.js` (these should be re-exported)
    - Ensure svg/index.ts re-exports these from themes (done in Plan 01, but verify the path)

    IMPORTANT: Keep the existing `svgResponse()` helper unchanged. Keep the anti-camo headers.
