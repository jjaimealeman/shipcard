    - Where the card embed/configurator currently lives (if any)

    Add a new "Theme Configurator" section to the dashboard HTML. Place it between the summary stats row and the activity chart section. The section has 3 sub-parts:

    **Sub-part A: Theme Swatch Grid (3x3)**
    - Display all 9 curated themes as clickable swatches in a 3-column grid
    - Each swatch: 80x60px rectangle with the theme's bg color, a "Aa" text sample in title color, a small dot in icon color, and the theme name below in small text
    - Active swatch has a highlighted border (use --orange or similar accent)
    - Default selected: catppuccin (or whatever the user's current embed uses)
    - Clicking a swatch sets `selectedTheme` state

    Embed the 9 theme palettes as a JS array directly in the HTML template (inline data, not a fetch). Format:
    ```js
    const THEMES = [
      { name: "catppuccin", label: "Catppuccin", bg: "#1e1e2e", border: "#313244", title: "#cdd6f4", text: "#a6adc8", icon: "#89b4fa" },
      // ... 8 more
    ];
    ```

    **Sub-part B: BYOT Custom Colors (PRO-gated)**
    - 5 hex input fields: bg, title, text, icon, border
    - Each field: label, text input with placeholder (e.g., "#1e1e2e"), inline validation indicator
    - PRO gate: Use a `isPro` Alpine.js state variable. For Phase 17, this can be a data attribute set on the dashboard container (the backend can set it based on `isUserPro()` KV check). If not PRO:
      - All 5 fields are `disabled` with reduced opacity
      - A lock icon and "Upgrade to PRO for custom colors" message overlays the section
      - Clicking the overlay links to shipcard.dev/upgrade (or a placeholder URL)
    - If PRO: fields are active. On input change:
      - Validate hex format (6 chars, valid hex digits). Show red border if invalid.
      - Run client-side contrast check (embed the `contrastRatio` formula in the Alpine.js data as a JS function — it's only ~10 lines). Check each field against bg. Show inline error per field if ratio < 3.0 (e.g., "Low contrast (2.1:1, min 3:1)").
      - Only update preview when all 5 fields are valid hex AND all pass contrast check.

    **Sub-part C: Live Preview + Embed Code**
    - Card preview: An `<img>` tag whose `src` is dynamically built:
      - Curated mode: `https://shipcard.dev/u/${username}?theme=${selectedTheme}&layout=${selectedLayout}`
      - BYOT mode: `https://shipcard.dev/u/${username}?bg=${bg}&title=${title}&text=${text}&icon=${icon}&border=${border}&layout=${selectedLayout}`
    - Use Alpine.js `x-effect` or `:src` binding to re-fetch on theme/color change
    - Add a small debounce (300ms) for BYOT inputs to avoid excessive fetches while typing
    - Layout selector: 3 radio buttons or tabs for classic/compact/hero (re-use existing layout selector if dashboard already has one, or add one)
    - Embed code: A `<textarea>` or `<code>` block below the preview showing the markdown embed code:
      ```
      ![ShipCard](https://shipcard.dev/u/{username}?theme={selectedTheme})
      ```
      Update dynamically as theme/layout changes. Include a "Copy" button.

    **Alpine.js state model:**
    ```js
    {
      selectedTheme: 'catppuccin',
      selectedLayout: 'classic',
      byotMode: false,  // true when user starts editing BYOT fields
      isPro: __IS_PRO__,  // replaced server-side
      byot: { bg: '', title: '', text: '', icon: '', border: '' },
      byotErrors: {},  // field -> error message
      // methods: selectTheme(), updateByot(), buildPreviewUrl(), buildEmbedCode(), copyEmbed()
    }
    ```

    **Server-side PRO flag:**
    In the dashboard route handler (the TypeScript code that serves the HTML), read PRO status:
    ```ts
    const isPro = await isUserPro(c.env.USER_DATA_KV, username);
    ```
    Replace `__IS_PRO__` placeholder in the HTML template with `true` or `false`.

    Import `isUserPro` from `../kv.js` in dashboard.ts.

    **CSS:**
    Add styles for the theme configurator that match the existing dashboard design:
    - `.theme-configurator` section wrapper with panel styling
    - `.theme-swatches` grid (3 columns, gap)

    - `.byot-section` with field grid
    - `.byot-field` with label, input, error text
    - `.byot-locked` overlay for free users
    - `.preview-section` with img container and embed code area
    - `.copy-btn` for the copy button

    Match the existing dashboard color variables (--bg, --surface, --border, --fg, --mid, --orange).
