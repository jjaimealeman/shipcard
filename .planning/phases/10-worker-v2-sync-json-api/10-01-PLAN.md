**Create `shiplog-worker/src/routes/syncV2.ts`** — mirrors existing sync.ts pattern:

```
import { Hono } from "hono";
import type { AppType } from "../types.js";
import { isValidSyncV2Body } from "../types.js";
import { authMiddleware } from "../auth.js";
import { putUserData, putTimeSeries, invalidateCardVariants, putCardCache } from "../kv.js";
import { renderCard } from "../svg/index.js";
```

**POST "/" handler** (behind authMiddleware):
1. Parse JSON body (try/catch, 400 on failure)
2. Validate with `isValidSyncV2Body(body)` — 400 on failure with `{ error: "Invalid v2 sync payload" }`
3. Verify `body.safeStats.username.toLowerCase() === authenticatedUsername.toLowerCase()` — 403 on mismatch
4. `await putUserData(env.USER_DATA_KV, username, body.safeStats)` — store stats (same key as v1)
5. `await putTimeSeries(env.USER_DATA_KV, username, body.timeSeries)` — store time-series (new key)
6. `const variantsInvalidated = await invalidateCardVariants(env.CARDS_KV, username)`
7. Re-render default card: `renderCard(body.safeStats, { theme: "dark", layout: "classic", style: "github" })` and `putCardCache`
8. `const syncedAt = new Date().toISOString()`
9. Return `{ ok: true, apiVersion: "v2", syncedAt, username, variantsInvalidated }`

Note: No separate DELETE on /sync/v2. The existing DELETE /sync already handles deletion (updated in Task 2 to also delete timeseries). One delete endpoint is cleaner.

**Update `shiplog-worker/src/index.ts`:**
1. Add import: `import { syncV2Routes } from "./routes/syncV2.js";`
2. Mount BEFORE v1 sync: `app.route("/sync/v2", syncV2Routes);` — must be above `app.route("/sync", syncRoutes)` so Hono matches the more specific `/sync/v2` path first.
3. Update the top JSDoc comment to include `POST /sync/v2` and the two new API GET routes (from Plan 02).
