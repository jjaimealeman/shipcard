  <action>
    Create shipcard-worker/src/routes/billing.ts as a Hono sub-app with 3 routes:

    **POST /checkout**
    - Protected by authMiddleware (user must be logged in)
    - Reads `interval` from request body: "month" (default) or "year"
    - Selects price ID: env.STRIPE_PRO_MONTHLY_PRICE_ID for month, env.STRIPE_PRO_ANNUAL_PRICE_ID for year
    - Looks up existing Stripe customer by username via `getSubscription(db, username)` — if exists, pass customer ID to avoid creating duplicate Stripe customers
    - Creates Stripe Checkout session:
      ```
      mode: 'subscription',
      line_items: [{ price: priceId, quantity: 1 }],
      subscription_data: { metadata: { username } },
      customer: existingSub?.stripe_customer_id || undefined,
      success_url: `${origin}/billing/welcome?session_id={CHECKOUT_SESSION_ID}`,
      cancel_url: `${origin}/u/${username}/dashboard`,
      ```
    - Returns `c.redirect(session.url!, 303)` (303 for POST->GET redirect)

    **POST /portal**
    - Protected by authMiddleware
    - Looks up subscription via `getSubscription(db, username)` — return 404 if none
    - Creates Stripe Customer Portal session:
      ```
      customer: sub.stripe_customer_id,
      return_url: `${origin}/u/${username}/dashboard`,
      ```
    - Returns `c.redirect(portalSession.url, 303)`

    **GET /welcome**
    - No auth required (Stripe redirects here after checkout)
    - Reads `session_id` query param
    - Serves a simple HTML page confirming PRO activation:
      - "Welcome to PRO!" heading
      - List of unlocked features: custom themes, custom slugs, AI insights
      - Link to dashboard: `/u/{username}/dashboard`
    - If session_id present, optionally retrieve session to get username for the dashboard link. If retrieval fails, show a generic "your dashboard" link without username.
    - Page should match existing design system (dark bg, Poppins font, orange accent)

    Mount in index.ts:
    - Add import: `import { billingRoutes } from "./routes/billing.js";`
    - Mount: `app.route("/billing", billingRoutes);`
    - Add to header docblock: billing route comments

    Use `c.req.header("origin") || c.req.header("x-forwarded-proto") + "://" + c.req.header("host")` for origin detection (or `new URL(c.req.url).origin`), matching how the codebase already handles origins.
  </action>
