    Create shipcard-worker/src/routes/billing.ts as a Hono sub-app with 5 routes.

    **Auth design rationale:** The dashboard is public (no login, no cookies, no Bearer tokens in the browser). Billing routes need to know WHO the user is. The solution: each billing action starts with a GitHub OAuth redirect. After OAuth callback verifies the user's identity, the route proceeds to the Stripe action. This is a stateless flow using a `state` parameter to carry intent through the OAuth redirect.

    **State encoding:** Use a base64url-encoded JSON string in the OAuth `state` param to carry the billing intent:
    ```typescript
    interface BillingState {
      action: 'checkout' | 'portal';
      interval?: 'month' | 'year'; // only for checkout
      nonce: string; // random string for CSRF-like protection
    }
    ```
    The state is NOT signed because we verify the user's identity via GitHub OAuth (the GitHub token exchange IS the auth). The nonce prevents replay attacks from cached URLs.

    **GET /checkout**
    - Reads `interval` query param: "month" (default) or "year"
    - Encodes BillingState as base64url JSON: `{ action: 'checkout', interval, nonce: crypto.randomUUID() }`
    - Redirects to GitHub OAuth authorize URL:
      ```
      https://github.com/login/oauth/authorize?client_id={GITHUB_CLIENT_ID}&redirect_uri={origin}/billing/checkout/callback&state={encodedState}&scope=read:user
      ```
    - Uses `new URL(c.req.url).origin` for origin detection

    **GET /checkout/callback**
    - GitHub redirects here with `code` and `state` query params
    - Decode `state` from base64url JSON, verify action === 'checkout'
    - Exchange `code` for GitHub access token via POST to `https://github.com/login/oauth/access_token` with client_id, client_secret, code (same pattern as existing /auth/exchange but server-side)
    - Fetch GitHub user via GET `https://api.github.com/user` with the access token to get `login` (username)
    - Select price ID: env.STRIPE_PRO_MONTHLY_PRICE_ID for 'month', env.STRIPE_PRO_ANNUAL_PRICE_ID for 'year'
    - Look up existing Stripe customer via `getSubscription(db, username)` to avoid duplicate Stripe customers
    - Create Stripe Checkout session:
      ```
      mode: 'subscription',
      line_items: [{ price: priceId, quantity: 1 }],
      subscription_data: { metadata: { username } },
      customer: existingSub?.stripe_customer_id || undefined,
      success_url: `${origin}/billing/welcome?session_id={CHECKOUT_SESSION_ID}`,
      cancel_url: `${origin}/u/${username}/dashboard`,
      ```
    - Redirect to `session.url` (302)
    - On any error (bad code, GitHub API failure), redirect to dashboard with `?error=auth_failed`

    **GET /portal**
    - Encodes BillingState as base64url JSON: `{ action: 'portal', nonce: crypto.randomUUID() }`
    - Redirects to GitHub OAuth authorize URL:
      ```
      https://github.com/login/oauth/authorize?client_id={GITHUB_CLIENT_ID}&redirect_uri={origin}/billing/portal/callback&state={encodedState}&scope=read:user
      ```

    **GET /portal/callback**
    - GitHub redirects here with `code` and `state` query params
    - Decode `state`, verify action === 'portal'
    - Exchange `code` for GitHub access token (same as checkout callback)
    - Fetch GitHub user to get username
    - Look up subscription via `getSubscription(db, username)` — if none, redirect to dashboard with `?error=no_subscription`
    - Create Stripe Customer Portal session:
      ```
      customer: sub.stripe_customer_id,
      return_url: `${origin}/u/${username}/dashboard`,
      ```
    - Redirect to `portalSession.url` (302)

    **GET /welcome**
    - No auth required (Stripe redirects here after checkout)
    - Reads `session_id` query param
    - Serves a simple HTML page confirming PRO activation:
      - "Welcome to PRO!" heading
      - List of unlocked features: custom themes, custom slugs, AI insights
      - Link to dashboard (if session_id present, retrieve session to get username for the link; if retrieval fails, show generic "your dashboard" link)
    - Page should match existing design system (dark bg, Poppins font, orange accent)

    **Helper functions (private to billing.ts):**
    ```typescript
    // Exchange GitHub OAuth code for access token
    async function exchangeGitHubCode(code: string, redirectUri: string, env: Env): Promise<string | null> {
      const resp = await fetch('https://github.com/login/oauth/access_token', {
        method: 'POST',
        headers: { 'Accept': 'application/json', 'Content-Type': 'application/json' },
        body: JSON.stringify({
          client_id: env.GITHUB_CLIENT_ID,
          client_secret: env.GITHUB_CLIENT_SECRET,
          code,
          redirect_uri: redirectUri,
        }),
      });
      if (!resp.ok) return null;
      const data = await resp.json() as { access_token?: string };
      return data.access_token || null;
    }

    // Fetch GitHub username from access token
    async function getGitHubUsername(accessToken: string): Promise<string | null> {
      const resp = await fetch('https://api.github.com/user', {
        headers: {
          Authorization: `Bearer ${accessToken}`,
          'User-Agent': 'shipcard-worker/1.0',
          Accept: 'application/vnd.github.v3+json',
        },
      });
      if (!resp.ok) return null;
      const user = await resp.json() as { login: string };
      return user.login || null;
    }

    // Encode/decode billing state
    function encodeBillingState(state: BillingState): string {
      return btoa(JSON.stringify(state)).replace(/\+/g, '-').replace(/\//g, '_').replace(/=/g, '');
    }
    function decodeBillingState(encoded: string): BillingState | null {
      try {
        const padded = encoded.replace(/-/g, '+').replace(/_/g, '/');
        return JSON.parse(atob(padded));
      } catch { return null; }
    }
    ```
