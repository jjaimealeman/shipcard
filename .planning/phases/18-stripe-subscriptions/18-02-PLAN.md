<task type="auto">
  <name>Task 1: Create webhook route handler</name>
  <files>shipcard-worker/src/routes/webhook.ts</files>
  <action>
    Create POST /stripe endpoint in a new Hono sub-app (webhookRoutes).

    The handler flow:
    1. Read raw body via `c.req.text()` (NOT json — Stripe needs the raw body for signature verification)
    2. Read `stripe-signature` header — return 400 if missing
    3. Call `stripe.webhooks.constructEventAsync(body, sig, env.STRIPE_WEBHOOK_SECRET, undefined, webCrypto)` — return 400 on failure. CRITICAL: use constructEventAsync, NOT constructEvent (sync version fails silently in Workers)
    4. Check idempotency: `isEventProcessed(db, event.id)` — if true, return 200 immediately
    5. Mark event processed: `markEventProcessed(db, event.id)` — do this BEFORE processing to prevent concurrent duplicates
    6. Switch on `event.type`:

       `checkout.session.completed`:
       - Cast data.object as Stripe.Checkout.Session
       - Retrieve the full subscription via `stripe.subscriptions.retrieve(session.subscription)` (required — session object doesn't have all fields)
       - Extract username from `subscription.metadata.username`
       - If no username in metadata, log warning and skip (return 200)
       - Call `upsertSubscription(db, username, { stripe_customer_id, stripe_subscription_id, status, plan: 'pro', current_period_end })`

       `customer.subscription.updated`:
       - Cast data.object as Stripe.Subscription
       - Extract username from `subscription.metadata.username`
       - If no username, try `getSubscriptionByStripeId(db, subscription.id)` to find the username
       - Call `upsertSubscription(db, username, ...)` with updated status and current_period_end

       `customer.subscription.deleted`:
       - Cast data.object as Stripe.Subscription
       - Find username from metadata or D1 lookup (same fallback as updated)
       - Call `markSubscriptionCanceled(db, username)`

       `invoice.payment_failed`:
       - Cast data.object as Stripe.Invoice
       - Extract subscription ID from `invoice.subscription`
       - Call `markPaymentFailed(db, subscriptionId)`

       `invoice.paid`:
       - Cast data.object as Stripe.Invoice
       - Extract subscription ID from `invoice.subscription`
       - Call `clearPaymentFailed(db, subscriptionId)`

    7. Return `c.json({ received: true }, 200)` for ALL events (including unhandled types)

    IMPORTANT: The entire handler should be wrapped in a try/catch that returns 200. Never return 5xx — Stripe retries on 5xx which causes duplicate processing.

    Use `import type` for Stripe types to minimize bundle size.
  </action>
  <verify>
    Run `cd shipcard-worker && npx tsc --noEmit` — compiles cleanly.
    Verify all 5 event types are handled in the switch statement.
  </verify>
  <done>
    Webhook handler verifies Stripe signatures asynchronously, processes all 5 lifecycle event types, writes to D1 via subscription helpers, and is idempotent via stripe_events table.
  </done>
</task>
