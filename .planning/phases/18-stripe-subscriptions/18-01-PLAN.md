<task type="auto">
  <name>Task 2: Create D1 schema + subscription query helpers</name>
  <files>
    shipcard-worker/src/db/schema.sql
    shipcard-worker/src/db/subscriptions.ts
  </files>
  <action>
    1. Create shipcard-worker/src/db/schema.sql with two tables:

       `subscriptions` table:
       - id INTEGER PRIMARY KEY AUTOINCREMENT
       - username TEXT NOT NULL UNIQUE
       - stripe_customer_id TEXT NOT NULL
       - stripe_subscription_id TEXT NOT NULL
       - status TEXT NOT NULL (values: active, past_due, canceled, incomplete, trialing)
       - plan TEXT NOT NULL DEFAULT 'pro'
       - current_period_end INTEGER (unix timestamp)
       - payment_failed_at INTEGER (unix timestamp, null when no failure)
       - created_at INTEGER NOT NULL DEFAULT (unixepoch())
       - updated_at INTEGER NOT NULL DEFAULT (unixepoch())

       Indexes:
       - idx_subscriptions_username ON subscriptions(username)
       - idx_subscriptions_stripe_sub_id ON subscriptions(stripe_subscription_id)

       `stripe_events` table (idempotency):
       - event_id TEXT PRIMARY KEY
       - processed_at INTEGER NOT NULL DEFAULT (unixepoch())

    2. Create shipcard-worker/src/db/subscriptions.ts with these functions:

       `isUserProFromD1(db, username)` — returns true if subscription status is 'active' or 'past_due' (grace period)

       `getSubscription(db, username)` — returns full subscription row or null

       `getSubscriptionByStripeId(db, stripeSubscriptionId)` — lookup by Stripe sub ID, returns row or null

       `upsertSubscription(db, username, data)` — INSERT OR REPLACE into subscriptions. Data includes stripe_customer_id, stripe_subscription_id, status, plan, current_period_end. Always updates updated_at.

       `markSubscriptionCanceled(db, username)` — UPDATE status to 'canceled', update updated_at

       `markPaymentFailed(db, stripeSubscriptionId)` — UPDATE payment_failed_at to current unixepoch() where stripe_subscription_id matches

       `clearPaymentFailed(db, stripeSubscriptionId)` — SET payment_failed_at = NULL where stripe_subscription_id matches

       `isEventProcessed(db, eventId)` — SELECT from stripe_events, returns boolean

       `markEventProcessed(db, eventId)` — INSERT into stripe_events

       Export a Subscription type matching the table columns.

       Use function-object style (no classes) matching codebase conventions. All functions take `db: D1Database` as first param.
  </action>
  <verify>
    Run `cd shipcard-worker && npx tsc --noEmit` — should compile cleanly.

    Verify subscriptions.ts exports all listed functions.
