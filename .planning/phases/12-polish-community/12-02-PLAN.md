  <name>Task 1: Mobile responsive CSS for dashboard</name>
  <files>shipcard-worker/src/routes/dashboard.ts</files>
  <action>
    Add mobile-first responsive CSS to the existing `<style>` block in dashboard.ts. This is CSS-only (no JS changes for layout).

    **Breakpoint strategy** (from CONTEXT.md):
    - Default (mobile): single column, everything stacks
    - 640px: 2-column for hero stats grid
    - 1024px: full desktop layout (existing styles become the desktop override)

    **Changes to make:**

    1. **Charts grid**: Change existing `.charts-grid` to default `grid-template-columns: 1fr` (single column). Move the current multi-column grid rules into `@media (min-width: 1024px)`.

    2. **Hero stats grid**: Default to `grid-template-columns: repeat(2, 1fr)` on mobile (4 hero stat cards in 2x2). At 1024px, restore `repeat(4, 1fr)`.

    3. **Panel body height**: Add explicit `height: 220px` on `.panel-body` (the chart canvas container) as the mobile default. At 1024px, set `height: 280px`. This prevents Chart.js canvas 0-height collapse when `maintainAspectRatio: false` is set. Chart.js `responsive: true` handles the rest automatically — do NOT add any JS resize logic.

    4. **Filter bar mobile**: Add a `<select>` element with class `mobile-range-select` and `x-model="$store.dashboard.range"` with options for 7d/30d/all. Use pure CSS to show/hide:
       - Default: `.mobile-range-select { display: block }`, `.btn-group { display: none }`
       - At 640px+: `.mobile-range-select { display: none }`, `.btn-group { display: flex }`
       This avoids Alpine init flash (Pitfall 4 from research).

    5. **Page container**: Ensure `.page` max-width and padding are responsive. On mobile, reduce side padding to 12px. At 1024px+, use the current padding.

    6. **Typography**: Reduce `.page h1` font-size on mobile (e.g., 1.5rem vs 2rem desktop). Panel titles can stay as-is.

    7. **Full-width panels**: On mobile, the calendar heatmap and activity overview panels should be `grid-column: 1 / -1` (full width) only on desktop where multi-column exists. On mobile single-column they naturally take full width.

    Do NOT use container queries or Alpine resize plugin — pure CSS media queries only.
    Do NOT add `chart.resize()` calls — Chart.js handles this via its own ResizeObserver.
  </action>

    Grep for `@media (min-width: 1024px)` in dashboard.ts — should find charts grid and panel height rules.
