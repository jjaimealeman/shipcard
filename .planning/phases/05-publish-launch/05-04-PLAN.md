    Run the npm dry run chain to verify the package works from a consumer's perspective:

    1. `cd shiplog && npm run build` — clean build (should already be done, but verify)
    2. `cd shiplog && npm pack` — creates shipcard-0.1.0.tgz
    3. `tar -tzf shipcard/shipcard-0.1.0.tgz | sort` — inspect contents, verify:
       - dist/ files present
       - data/pricing-snapshot.json present
       - No src/, node_modules/, or .gitignore
    4. `npm install -g ./shipcard/shipcard-0.1.0.tgz` — global install from tarball
    5. `shipcard --help` — verify CLI bin entry works
    6. `shipcard --version` — verify version output
    7. `shipcard-mcp --version` or just verify it starts (it's an MCP stdio server, so it may hang waiting for input — use timeout)
    8. `npm uninstall -g shipcard` — clean up global install
    9. Test npx path: `npx ./shipcard/shipcard-0.1.0.tgz summary` — verify npx works from tarball

    If any step fails, diagnose and fix. Common issues:
    - Missing shebang -> check source files for `#!/usr/bin/env node` at line 1
    - Permission denied -> check chmod 755 in build script
    - Module not found -> check that imports use .js extensions
    - data/ missing -> check files field in package.json

<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
    Full package dry run is complete. The tarball has been verified locally:
    - `shipcard` CLI works after global install
    - `shipcard-mcp` server binary works
    - `npx shipcard` works from tarball
    - Tarball contains dist/, data/, package.json, LICENSE

    Before npm publish, the Worker needs to be deployed and external services configured.
  </what-built>
  <how-to-verify>
    **Step 1: Create KV namespaces**
    ```bash
    cd shiplog-worker
    npx wrangler kv namespace create CARDS_KV
    npx wrangler kv namespace create USER_DATA_KV
    ```
    Copy the returned IDs and paste them into wrangler.jsonc (replacing the placeholder IDs).

    **Step 2: Create GitHub OAuth App**
    Go to: GitHub Settings -> Developer Settings -> OAuth Apps -> New OAuth App
    - Application name: ShipCard
    - Homepage URL: https://shipcard.dev
    - Authorization callback URL: https://shipcard.dev/auth/callback
    Copy the Client ID and Client Secret.

    **Step 3: Set Worker secrets**
    ```bash
    cd shiplog-worker
    npx wrangler secret put GITHUB_CLIENT_ID    # paste Client ID
    npx wrangler secret put GITHUB_CLIENT_SECRET # paste Client Secret
    npx wrangler secret put TOKEN_SECRET         # paste a random 32-char hex string
    ```

    **Step 4: Deploy Worker**
    ```bash
    cd shiplog-worker
    npx wrangler deploy
    ```
    Verify: `curl https://shipcard.dev/` should return `{"name":"shipcard","status":"ok",...}`

    **Step 5: Test card endpoint**
    `curl -I https://shipcard.dev/u/testuser` — should return 200 with Content-Type: image/svg+xml

    **Step 6: npm publish**
    ```bash
    cd shiplog
    npm login          # if not already logged in
    npm publish --access public
    ```
    (Will prompt for OTP if 2FA is enabled on your npm account)

    **Step 7: Verify registry install**
    ```bash
    npm install -g shipcard
    shipcard --help
    ```

    Type "done" when all steps are complete, or describe any issues.
  </how-to-verify>
