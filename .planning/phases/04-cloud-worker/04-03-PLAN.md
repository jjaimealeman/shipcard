  <action>
    **shiplog/src/cli/safestats.ts:** Create a `toSafeStats(result: AnalyticsResult, username: string): SafeStats` function. This is the privacy boundary on the CLI side — it converts AnalyticsResult to a shape safe for cloud transmission:
    - `username` from parameter
    - `totalSessions` from `result.summary.totalSessions`
    - `totalTokens` from `result.summary.totalTokens` (input, output, cacheCreation, cacheRead)
    - `totalCost` from `result.summary.totalCost`
    - `modelsUsed` from `result.summary.modelsUsed`
    - `projectCount` from `result.summary.projectsTouched.length` (count only, NOT names)
    - `toolCallSummary` from `result.summary.toolCallSummary`
    - `pricingVersion` from `result.summary.pricingVersion`

    Define the `SafeStats` interface locally in this file (mirrors Worker's types.ts definition). Do NOT import from the Worker package.

    CRITICAL: This function must NEVER include `projectsTouched` (the names array), `byProject` (per-project breakdown), `meta.dateRange`, `meta.filesRead`, `meta.linesSkipped`, or any file path.

    **shiplog/src/cli/config.ts:** Update or create config management for `~/.shiplog/config.json`:
    - `interface ShiplogConfig { username?: string; token?: string; workerUrl?: string }`
    - `loadConfig(): Promise<ShiplogConfig>` — reads `~/.shiplog/config.json`, returns empty object if not found
    - `saveConfig(config: Partial<ShiplogConfig>): Promise<void>` — merges with existing config and writes
    - `getConfigPath(): string` — returns `~/.shiplog/config.json`
    - Default `workerUrl`: `https://shiplog.workers.dev`

    Use `node:fs/promises` and `node:os` (homedir). Create `~/.shiplog/` directory if it doesn't exist (`mkdir -p` equivalent with `recursive: true`).

    **shiplog/package.json:** Add `@octokit/auth-oauth-device` ^7.x as a dependency. Run `npm install`.

    **shiplog/src/cli/commands/login.ts:** Implement the `shiplog login` command:
    1. Import `createOAuthDeviceAuth` from `@octokit/auth-oauth-device`
    2. Create auth with `clientType: 'oauth-app'`, hardcoded public `clientId` (use a placeholder constant `SHIPLOG_GITHUB_CLIENT_ID` — the user will create the OAuth App and fill this in), scopes: `['read:user']`
    3. `onVerification` callback: print the verification URL and user code to stderr, attempt to open browser with the URL (use a simple platform-aware open: `xdg-open` on Linux, `open` on macOS, `start` on Windows — reuse the pattern from `shiplog/src/card/preview.ts` if one exists)
    4. Await `auth({ type: 'oauth' })` to get GitHub token
    5. Fetch `https://api.github.com/user` with the GitHub token to get username
    6. POST to `{workerUrl}/auth/exchange` with `{ githubToken, username }` to get Worker token
    7. Save `{ username, token }` to config via `saveConfig()`
    8. Print success message: "Logged in as {username}"

    Error handling: If device flow times out (15 min), print helpful message. If Worker exchange fails, print the HTTP status and error.

    **shiplog/src/cli/args.ts:** Add `login` and `sync` to the known commands. Add `--delete` flag recognition for sync.

    **shiplog/src/cli/commands/sync.ts:** Implement the `shiplog sync` command:

    `shiplog sync` (default — interactive sync with configurator):
    1. Load config — if no token, print "Run `shiplog login` first" and exit
    2. Run the analytics engine to get AnalyticsResult (same as summary/costs commands)
    3. Convert to SafeStats via `toSafeStats(result, config.username)`
    4. Print a summary of what will be sent to the terminal (stat names + values, NO raw data):
       ```
       Sync preview:
         Sessions: 142
         Tool Calls: 1,847
         Models: claude-opus-4-6, claude-sonnet-4-6
         Projects: 5 (names hidden)
         Est. Cost: ~$127.50
       ```
    5. Encode SafeStats as base64 JSON in a URL hash fragment
    6. Open browser to `{workerUrl}/configure#{base64data}`
    7. Print: "Configure your card in the browser, then run the command shown there."
    8. Print: "Or sync with defaults: shiplog sync --confirm"

    `shiplog sync --confirm` (non-interactive sync with current/default settings):
    1. Load config, run engine, convert to SafeStats
    2. POST to `{workerUrl}/sync` with bearer token and SafeStats body
    3. On success, print card URL and embed snippets:
       ```
       Card synced! View at:
         https://shiplog.workers.dev/card/{username}

       Markdown:
         ![ShipLog Stats](https://shiplog.workers.dev/card/{username})

       HTML:
         <img src="https://shiplog.workers.dev/card/{username}" alt="ShipLog Stats" />

       Customize appearance with query params:
         ?theme=dark&layout=hero&style=branded
       ```

    `shiplog sync --delete`:
    1. Load config, check for token
    2. DELETE to `{workerUrl}/sync` with bearer token
    3. Print: "All data removed for {username}"

    **shiplog-worker/src/routes/configure.ts:** Serve a self-contained HTML page at GET /configure.

    The configurator is a single HTML page with inline CSS and JS (no external dependencies). It should:

    1. On load, read stats from URL hash fragment (`#base64encodedJSON`), decode and parse
    2. Display the stats in a readable format with toggle checkboxes for each stat (sessions, toolCalls, models, projects, cost)
    3. Provide dropdowns for: layout (classic/compact/hero), style (github/branded/minimal), theme (dark/light)
    4. Show a live SVG preview that updates when toggles or dropdowns change — render the preview by calling the Worker's own card endpoint with the appropriate query params. Use an `<img>` tag pointing at `/card/{username}?theme=...&layout=...&style=...` BUT since the user hasn't synced yet, the preview should use an `<iframe>` or inline SVG approach. Simpler: render a static preview using the stats data directly in the browser. Build a simplified SVG template in JS that mirrors the card look.
    5. Generate the CLI command: `shiplog sync --confirm --hide sessions,cost --layout hero --theme dark --style branded` (based on current selections)
    6. Show ready-to-copy Markdown and HTML embed snippets with the appearance params
    7. Persist settings in localStorage keyed by username

    Design: Plain, utilitarian. The card is the star, not the configurator chrome. Light gray background, simple controls, large preview area. No framework, no build step.
