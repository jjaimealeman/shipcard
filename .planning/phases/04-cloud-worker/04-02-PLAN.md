<objective>
Add authentication and the POST /sync endpoint to the Worker, completing the write path for stats.

Purpose: This is how stats get from the user's machine to the cloud. The auth exchange endpoint converts a GitHub OAuth token into a Worker-issued bearer token. The sync endpoint accepts SafeStats payloads, validates the privacy boundary (no paths, no content), stores data, invalidates stale cached cards, and re-renders the default card variant synchronously to avoid KV eventual consistency issues.

Output: Working auth middleware, token exchange endpoint, and sync endpoint. After this plan, the full Worker read+write cycle works: sync stats, then GET /card/:username returns updated card.
</objective>

  <action>
    **src/auth.ts:** Hono middleware that validates bearer tokens:
    1. Extract `Authorization` header, check it starts with `Bearer `
    2. Extract token value (slice off "Bearer ")
    3. Look up `token:{token}:username` in USER_DATA_KV via `getTokenUsername()`
    4. If not found, return `c.json({ error: 'Invalid or expired token' }, 401)`
    5. If found, set username on Hono context: `c.set('username', username)`
    6. Call `await next()`

    Use Hono's `MiddlewareHandler` type with the correct generic: `MiddlewareHandler<{ Bindings: Env; Variables: { username: string } }>`.

    Update `src/types.ts` to add `Variables` to AppType:
    ```
    export type AppType = { Bindings: Env; Variables: { username: string } }
    ```

    **src/routes/auth.ts:** Export a Hono sub-app with one endpoint:

    `POST /exchange` â€” Token exchange:
    1. Accept JSON body: `{ githubToken: string, username: string }`
    2. Verify the GitHub token by calling `https://api.github.com/user` with it
    3. Compare returned `login` with the claimed `username` â€” reject if mismatch (prevents impersonation)
    4. Generate a Worker-issued opaque token: `crypto.randomUUID()`
    5. Store in KV via `putToken(env.USER_DATA_KV, token, username)` with 1-year TTL
    6. Return `{ token, username }` with status 200

    Error cases:
    - Missing/invalid GitHub token: 401
    - Username mismatch: 403 with `{ error: 'Username mismatch' }`
    - GitHub API error: 502 with `{ error: 'GitHub API unavailable' }`

    Use `User-Agent: shiplog-worker/1.0` header when calling GitHub API (required by GitHub).
  </action>

  </verify>

    **src/types.ts additions:** Add a `isValidSafeStats(payload: unknown): payload is SafeStats` type guard function. It MUST check:
    1. `payload` is a non-null object
    2. `username` is a string
    3. `totalSessions` is a number >= 0
    4. `totalTokens` is an object with `input`, `output`, `cacheCreation`, `cacheRead` as numbers
    5. `totalCost` is a string
    6. `modelsUsed` is an array of strings
    7. `projectCount` is a number >= 0
    8. `toolCallSummary` is a Record (object with string keys and number values)
    9. `pricingVersion` is a string

    CRITICAL privacy checks â€” reject if ANY of these are present:
    - Any field named `path`, `paths`, `filePath`, `projectsDir`, `cwd`
    - Any field named `content`, `rawContent`, `jsonl`
    - Any field named `projectsTouched` or `projectNames` (names are private; only `projectCount` is allowed)
    - Any field containing a string that looks like a file path (starts with `/` or `~`)

    These checks enforce CLOUD-04: only numeric aggregates + username reach the cloud.

    **src/routes/sync.ts:** Export a Hono sub-app:

    `POST /` (mounted at /sync, so effective path is POST /sync):
    1. Apply `authMiddleware` to the route
    2. Parse JSON body
    3. Validate with `isValidSafeStats()` â€” if invalid, return 400 with `{ error: 'Invalid SafeStats payload' }`
    4. Verify `body.username` matches the authenticated username from middleware (`c.get('username')`) â€” reject 403 if mismatch
    5. Store user data: `putUserData(env.USER_DATA_KV, username, body)`
    6. Invalidate all cached card variants: `invalidateCardVariants(env.CARDS_KV, username)`
    7. Re-render the default card variant (theme=dark, layout=classic, style=github) and store in cache via `putCardCache()` â€” this avoids stale reads from KV eventual consistency (see Pitfall 2 in research)
    8. Return `{ ok: true, username, variantsInvalidated: count }`

    `DELETE /` (mounted at /sync, so effective path is DELETE /sync):
    1. Apply `authMiddleware`
    2. Delete all user data and card variants: `deleteAllUserData(env.USER_DATA_KV, username)` + `invalidateCardVariants(env.CARDS_KV, username)`
    3. Return `{ ok: true, deleted: true, username }`
    Note: Does NOT delete the auth token (per research decision #4). User can still re-sync later.

    **src/index.ts updates:** Register the new routes:
    ```
    app.route('/auth', authRoutes)
    app.route('/sync', syncRoutes)
    ```
