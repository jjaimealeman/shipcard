2. **CSS** — Use CSS custom properties for the Anthropic brand palette:

   - `--fg: #faf9f5` (light text)
   - `--mid: #b0aea5` (mid gray)
   - `--light: #e8e6dc` (light gray)
   - `--orange: #d97757` (accent)
   - `--blue: #6a9bcc` (accent)
   - `--green: #788c5d` (accent)
   - Poppins for headings (font-weight 600/700), Lora for body text (font-weight 400)
   - Fully responsive — mobile-first, sections stack vertically
   - Style the configurator controls (toggle buttons, select dropdowns, checkboxes)
   - Terminal-styled code blocks with dark background and monospace font
   - Tab component styles for Markdown/HTML snippet switcher

3. **Hero section** —
   - Headline: "See what you shipped with Claude Code" (Poppins 700)
   - Subtext: sessions, tokens, cost, one embeddable card (Lora 400)
   - Clean, centered layout with generous vertical spacing

4. **Card configurator section** —
   - Username text input (defaults to "jjaimealeman", placeholder "GitHub username")
   - Theme toggle: dark / light (button group)
   - Layout toggle: classic / compact / hero (button group)
   - Style toggle: flat / shadow / border (button group)
   - Hide stats: individual checkboxes for sessions, toolCalls, models, projects, cost
   - Card preview container (div where SVG gets injected via innerHTML)
   - Reset button to restore all defaults
   - Tabbed code block below the preview:

     - Tab 2: HTML embed snippet
     - Each tab has a Copy button
     - Snippets update live when configurator state changes

5. **Quickstart section** —
   - "Get it going in 30 seconds" heading
   - 3 terminal-styled code blocks with copy buttons:

   - npm package link (https://www.npmjs.com/package/shipcard)
   - MIT license badge/text
   - MCP server mention
   - "Made by" attribution

**Embedded JavaScript (critical rules):**

ALL client-side JS must use `var`/`function` declarations with single-quoted strings. NO template literals inside the embedded script (they conflict with the outer template literal). NO `${}` expressions. Follow configure.ts conventions exactly.

- `refreshCard()` — fetches `/u/{username}?{params}` and injects SVG into preview container via innerHTML. Debounce username input at 400ms.
- `buildQs(state)` — builds query string from configurator state. Handle `hide` param by appending multiple `&hide=key` entries (not URLSearchParams).
- `getState()` / `resetState()` — read current toggle values / reset to defaults.
- `updateSnippets(username, qs)` — updates Markdown and HTML tab content with current card URL.
- `copyText(text, btn)` — clipboard copy with fallback (navigator.clipboard.writeText + execCommand fallback). Same pattern as configure.ts.
- Tab switching — CSS class toggle between Markdown and HTML tabs.
- Event listeners on all controls: call refreshCard() on change. Debounce only the username input.
- On page load: call refreshCard() to load the default card.

**Empty state:** When a username has no card, the `/u/:username` endpoint returns a valid placeholder SVG (HTTP 200). Inject it as-is — the placeholder IS the empty state.
