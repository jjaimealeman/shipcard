# Milestone v2.0: Themes + Monetization

**Status:** SHIPPED 2026-03-29
**Phases:** 16-21
**Total Plans:** 22

## Overview

Turn ShipCard into a sustainable business — curated themes make cards worth sharing, Stripe subscriptions create recurring revenue, PRO features justify the upgrade, and AI insights make the dashboard indispensable for power users.

## Phases

### Phase 16: Agent-Agnostic Architecture

**Goal:** The parser engine works with any agent's JSONL format through a clean adapter interface, with zero behavior change for Claude Code users.
**Depends on:** Phase 15
**Requirements:** ARCH-01, ARCH-02, ARCH-03
**Success Criteria:**
  1. All existing `shipcard` commands produce identical output before and after the refactor
  2. A `SourceAdapter` interface exists and the existing Claude Code parser is registered as `ClaudeCodeAdapter`
  3. The engine processes `ParsedMessage[]` from any adapter — no parser-specific logic leaks into engine code
  4. Adding a hypothetical second adapter requires zero changes to engine or CLI code

- [x] 16-01-PLAN.md — Rename shiplog/ to shipcard/ and update all references
- [x] 16-02-PLAN.md — Create SourceAdapter interface, ClaudeCodeAdapter, wire into engine, version 2.0.0

## Milestone Summary

**Key Decisions:**
- D1 over KV for subscription state (strong consistency required for billing)
- GET + GitHub OAuth redirect flow for billing (dashboard is public, no Bearer tokens)
- WCAG 3:1 contrast ratio for card themes (UI component standard, not body text)
- Function-object pattern for adapters (matches codebase conventions)
- catppuccin as default theme for new users (best visual default)
- Two-write pattern for AI insights (stats immediately, narrative via waitUntil)
- All Clack imports centralized through clack.ts TTY-guard module

**Issues Resolved:**
- Adapter architecture enables future multi-agent support without engine changes
- D1 migration eliminates KV eventual-consistency risk for billing state
- BYOT cache skip prevents unbounded KV growth from arbitrary color combinations

**Issues Deferred:**
- BYOT saved presets (named theme configs stored per user)
- Support for Codex CLI and Gemini CLI JSONL formats
- Team dashboards with cost allocation
- Weekly email digest of AI insights

**Technical Debt Incurred:**
- syncV2 default card pre-render uses v2 key format (fixed during audit)
- 18 visual/runtime tests need human verification with live Worker + Stripe
- Orphaned listAdapters() export removed during audit
