# Milestone v1.1: Dashboard Enhancement

**Status:** SHIPPED 2026-03-28
**Phases:** 13-15
**Total Plans:** 5

## Overview

Make the dashboard the reason people adopt ShipCard ‚Äî richer per-project breakdowns, today's activity hero, and meaningful sort dimensions. Three phases: enrich the data pipeline with per-project stats, add hero sections (Today's Activity + Peak Days), and wire sort toggles for slicing project performance by any metric.

### Phase 14: Hero Section

**Goal**: The dashboard opens with a vivid today-vs-yesterday snapshot and a Peak Day trophy card.
**Depends on**: Phase 13
**Plans**: 2 plans

Plans:
- [x] 14-01-PLAN.md ‚Äî Today's Activity section with 4 metrics and direction indicators vs yesterday
- [x] 14-02-PLAN.md ‚Äî Peak Day per-metric cards with date, value, and project name

**Details:**
- 14 Alpine store computed getters for today/yesterday using local calendar day boundaries
- Direction indicators: orange up, blue down (not red/green alarm colors)
- Peak Days: 4 per-metric all-time records (messages, sessions, tokens, cost)
- Peak getters scan all days (not filteredDays) ‚Äî immune to range selector
- `_fmtShortDate` uses noon (T12:00:00) to avoid timezone shift

**Plans**: 1 plan

## Milestone Summary

**Key Decisions:**
- byProject is optional on DailyStats ‚Äî backward compatible with all existing consumers
- Worker mirrors CLI types exactly (no cross-package import)
- Calendar day boundaries for "today" (not rolling 24h)
- Neutral direction indicators (orange/blue, not red/green)
- Peak Days show 4 per-metric cards (not single combined card)
- Sort toggle updates chart in-place (avoids destroy/recreate canvas bug)
- Free tier project cap is 5 (not 10)

**Issues Resolved:**
- userMessages per day was hardcoded 0 ‚Äî fixed with real UserEntry timestamp data
- Sort toggle chart blank after first toggle ‚Äî fixed with in-place chart.update()

**Issues Deferred:**
- Per-project userMessages hardcoded 0 (UserEntry JSONL has no project association ‚Äî architectural limitation)
- Sort toggle buttons remain interactive on legacy data (minor UX gap, not a code error)

**Technical Debt Incurred:**
- None significant
