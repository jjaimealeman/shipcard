## Tech Debt

### Phase 13: Data Pipeline + Cleanup
- Per-project `userMessages` hardcoded to `0` — UserEntry JSONL entries have no project association field. Day-level userMessages is correct. This is an architectural limitation of the JSONL format, not a stub.

### Phase 15: Project Activity
- Sort toggle buttons remain interactive on legacy data (no `byProject`). Selecting tokens/sessions/cost yields an empty chart because legacy entries only have a days-active proxy. The "Days active" label correctly shows when `hasByProject` is false, and the filter correctly handles undefined fields. Not a code error — a minor UX gap where buttons could be visually disabled for legacy users.
